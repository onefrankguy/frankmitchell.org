Let’s talk about me

I’m a build engineer

The first build system I ever wrote was a home grown AppleScript that pushed text files up to a web server. I used it to post lecture notes online, which resulted in other students not showing up for class. It taught me that technology can change people’s lives.

The second build system I wrote was for Chief Architect.1 It started out as a simple Ruby script for generating Windows installers. Three years later, it was a Visual Studio integrated, RSS file generating, 3D model packaging, content distribution system. It taught me that software needs to be flexible.

I ship things

The most recent thing I shipped was Prolix,2 a tweetable word search game for the iPhone and iPod touch. It works on Android too, and in your web browser. My wife found a rotation bug with the letters a week after it launched. It taught me that real users make the best testers.

Before that, I collaborated with a graphic artist to create a deck of yoga cards.3 It was a struggle to describe human poses and printed colors in emailed words. It taught me that good communication will get your further than anything else.

I have a philosophy

Computers can make the world a better place. We’re not there yet. There are still too many sharp edges and clunky interfaces; not enough engineers design what they build or can build what they design.

We’ll know we’ve gotten there when computers start showing up in kitchens and bathrooms and bedrooms. When they start fading into the background of our lives. When they stop looking and feeling like computers and start feeling like extensions of our selves.

I think that’s where litl’s going.

Let’s talk about you

You’re an awesome company

You’re litl. You make a webbook that makes family life better. You’re working on a web-connected TV product. You’re looking for a build engineer. And you have a philosophy that makes me shout, “Yay! Someone else gets how a computer should work.”

You need an awesome engineer

You need someone who understands that deployment isn’t just a Python script running on a build machine. That it’s a process that has to be designed hand-in-hand with a product. Someone who’s flexible enough to balance the needs of development, QA, and customer service. An engineer who can say, “No. We’re not putting that in the build system,” and then come up with a creative solution for where it should go.

I’ll be honest. Working at litl is my dream job. You’re one of the few companies that’s built a computer and OS from the ground up. You’re high profile, and you’ve got a lot of competition. There’s going to be huge pressure to make sure deployment gets handled right. Updates aren’t opt-in, so one bad packet could take down every litl out there. You’re going to need somebody who’s creative under pressure, can communicate well and can get things done.

I’m that guy

I spent almost as much time on this letter pushing pixels and setting type as I did editing content. I did so, because I believe the web is the future, and user experience is important. Products like the litl, that merge the web with beautiful design, are the direction computing has to go.

litl’s a company where I can learn new things. Where I can cultivate my passion for better user experiences. Where I can help craft products that are technologically rich and simple enough for my grandparents to use. It’s a place where I can grow.

I want to be a litl engineer.
